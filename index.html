BluePrint Privacy Policy

Last Updated: May 2026

Overview

BluePrint is a student accountability and attendance system that uses NFC scanning and fingerprint authentication to securely verify user identity and record attendance events.

Information We Collect

BluePrint may collect the following types of data:

  • Student or User ID (internal identifier for authentication and attendance tracking)
  • NFC scan data (unique tag identifiers used for check-in/out)
  • Fingerprint authentication data (stored as encrypted biometric templates, not raw fingerprint images)
  • Device and system logs (timestamps, scan events, and system status information)

How We Use Information

We use collected data strictly for:

  • Attendance tracking and verification
  • Authenticating authorized users
  • Maintaining system integrity and preventing misuse
  • Improving reliability of the BluePrint system

Data Storage and Security

Data collected by BluePrint is transmitted securely to a backend system and/or stored locally on authorized hardware devices. Communication between components is secured using standard encrypted network protocols where applicable.

Data Sharing

BluePrint does not sell user data and does not share personal data with third-party advertisers or data brokers. Data is only accessible to authorized system components and administrators responsible for operating the system.

Tracking

BluePrint does not use user data for tracking, advertising, or cross-app user profiling. No data is shared with third-party advertising networks or used for targeted advertising purposes.

User Rights

Users may request access to or deletion of their data by contacting the system administrator or organization operating BluePrint.

Contact

For privacy-related questions, contact the BluePrint system administrator or project maintainer.
